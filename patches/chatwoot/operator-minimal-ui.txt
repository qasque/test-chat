Минимальный операторский UX в форке Chatwoot (qasque/chatwoot-custom)
=====================================================================

Цель: в веб-дашборде оставить только сценарии входа оператора и работы с
диалогами (inbox / conversations). Убрать лишние пункты бокового меню:
настройки аккаунта, отчёты, кампании и т.д. — по политике продукта.

Где править (актуально для ветки develop, см. docs/CHATWOOT-SOURCE.txt)
-----------------------------------------------------------------------
- Основная боковая панель (Vue 3): ориентир upstream —
  app/javascript/dashboard/components-next/sidebar/Sidebar.vue
  и соседние SidebarGroup.vue, SidebarProfileMenu.vue, маршруты в sidebar.

Реализация в форке (develop)
----------------------------
В `app/javascript/dashboard/components-next/sidebar/Sidebar.vue`:
композабл `useAdmin()` — если роль **не** `administrator`, в навигации
остаются только группы **Inbox** и **Conversation** (все подпункты диалогов:
все беседы, упоминания, необслуженные, быстрые ответы, заметки, папки,
команды, каналы, метки). Остальные разделы сайдбара — только у администратора.

Подход при доработках
---------------------
1) Не ломать мобильный flyout sidebar — тот же `Sidebar.vue`.

2) Прямые URL к скрытым разделам могут оставаться доступны по правам
   бэкенда; при необходимости усилить — отдельно в `routes` (permissions).

3) После правок: сборка фронта / smoke-логин под agent и administrator.

Мобильные приложения (android-chat / ios-chat)
----------------------------------------------
Репозитории не в этом монорепо. Параллельно с форком Chatwoot:
- Нижнее/боковое меню: оставить только экраны «оператор» — вход
  (POST …/mobile/v1/auth/login через bridge) и список/экран диалога.
- Скрыть пункты вроде статуса стека, настройки бота, «ещё» для роли
  operator, если такие есть (или завести build flavor / dart-define).

Портал periliv (apps/web)
-------------------------
Режим только диалогов без меню портала: в .env задайте
  PORTAL_UI_MODE=operator
и пересоберите сервис portal (config.js получит флаг при старте контейнера).
Локально: VITE_PORTAL_UI_MODE=operator npm run dev.
