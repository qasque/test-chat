# Синхронизация с закрытым репозиторием

Краткий план: ты выдаёшь доступ к **приватному** репозиторию на GitHub (или другом хосте), после чего в этой среде можно **клонировать или подтянуть** всё содержимое командой `git`.

## Целевой репозиторий для «слияния» этого монорепо

Локальная копия этого проекта исторически трекает **`origin`** → `qasque/test-chat`. **Новый канонический remote для выкладки всего стека** — пустой (или только что созданный) репозиторий:

| Поле | Значение |
|------|----------|
| **URL (SSH)** | `git@github.com:qasque/periliv.git` |
| **URL (HTTPS)** | `https://github.com/qasque/periliv.git` |
| **Ветка по умолчанию** | `main` (как в текущем локальном репо) |
| **Назначение** | В `periliv` заливается **весь** этот монорепозиторий; `test-chat` можно оставить как прошлый `origin` или позже переназначить — см. шаги ниже. |

Секреты (токены, пароли) **не** вписывать в этот файл и **не** коммитить в git.

---

## Что сделать тебе (один раз)

1. Создать **пустой или существующий** приватный репозиторий и залить туда нужный код (или решить, что источник правды — уже этот репо, и доступ нужен только для копии на другой машине).
2. Выдать доступ одним из способов:
   - **SSH:** добавить свой публичный ключ в GitHub → клонирование вида `git clone git@github.com:ORG/REPO.git`
   - **HTTPS + токен:** Personal Access Token с правами `repo` → `git clone https://github.com/ORG/REPO.git` (логин + токен как пароль)
   - **Collaborator:** пригласить аккаунт, с которого будет выполняться `git clone` / `git pull`

## Для ассистента (ИИ) на **macOS**: что делать с `periliv`

Ниже — сценарий «локальная папка с этим репо уже есть, на GitHub создан **пустой** `qasque/periliv`, нужно **первый push** всей истории». Команды выполнять в **Terminal** (или встроенном терминале Cursor) из **корня** клона.

### 0. Предпосылки на Mac

- Установлен **Git** (через Xcode Command Line Tools: `xcode-select --install`, или `brew install git`).
- Доступ к GitHub по **SSH** (рекомендуется): ключ добавлен в [GitHub → SSH keys](https://github.com/settings/keys). При необходимости добавить ключ в агент (часто нужно после перезагрузки):

  ```bash
  eval "$(ssh-agent -s)"
  ssh-add --apple-use-keychain ~/.ssh/id_ed25519
  ```

  (имя файла ключа замени на свой, например `id_rsa`.)

- Проверка: `ssh -T git@github.com` — должно приветствовать пользователя без запроса пароля (или с разовым подтверждением keychain).

### 1. Убедиться, что ты в нужной ветке

```bash
cd /path/to/this/monorepo
git status
git branch --show-current   # ожидается main
```

### 2. Добавить remote `periliv` (не трогая старый `origin`)

```bash
git remote add periliv git@github.com:qasque/periliv.git
git remote -v
```

Если remote с именем `periliv` уже ошибочно указывает куда-то ещё: `git remote remove periliv` и повтори `add`.

### 3. Первый push в пустой репозиторий

```bash
git push -u periliv main
```

- Если на GitHub в `periliv` **уже** есть коммиты (например, README при создании репо), перед push нужно либо **force** (осторожно, перезапишет удалённую историю), либо **pull с merge/rebase** — уточни у владельца, что на GitHub «источник правды». Для **пустого** репо достаточно команды выше.

### 4. (Опционально) сделать `periliv` основным `origin`

Если решили больше не пушить в `test-chat`:

```bash
git remote rename origin old-test-chat
git remote rename periliv origin
git push -u origin main
```

Или оставить два remote: `origin` = старый, `periliv` = новый — тогда дальше пушить так: `git push periliv main`.

### 5. HTTPS вместо SSH (если SSH недоступен)

```bash
git remote add periliv https://github.com/qasque/periliv.git
git push -u periliv main
```

Git спросит учётные данные: для GitHub обычно **username + Personal Access Token** (не пароль от аккаунта).

## Что сделать здесь (после доступа) — общий случай «только клон»

В каталоге, куда нужно скачать проект:

```bash
git clone git@github.com:qasque/periliv.git
cd periliv
git status
```

Если репозиторий уже склонирован и нужно только обновить:

```bash
git pull --ff-only
```

## Для ассистента (ИИ) — общее

- Полный текст переписок из Cursor **в репозиторий не входит**; для контекста тем см. **`docs/CURSOR-AGENTS-CHAT-INDEX.md`** и **`docs/REPO-CONTEXT.md`**.
- После того как пользователь подтвердил доступ и при необходимости передал **только URL** (без токенов в чат), имеет смысл выполнить `git clone` / `git pull` / `git push` **в той среде, где уже настроена аутентификация** (SSH-agent, macOS Keychain, credential manager и т.д.).
- На **macOS** для первого push в `periliv` используй раздел **«Для ассистента (ИИ) на macOS»** выше; текущая локальная ветка по умолчанию в этом репо — **`main`**, дополнительные remotes **`android-chat`** / **`ios-chat`** пушить в `periliv` **не нужно** (это другие репозитории).
