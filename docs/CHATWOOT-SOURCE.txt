Chatwoot в этом стеке — исходники в папке chatwoot/ (не в git test-chat).

Первый раз на сервере (из корня test-chat):

  chmod +x scripts/clone-chatwoot.sh scripts/update-chatwoot-stack.sh
  ./scripts/clone-chatwoot.sh
  docker compose build rails
  docker compose build sidekiq
  docker compose up -d

Обновить код форка (фронт + бэк) после git push в репозиторий Chatwoot:

  ./scripts/update-chatwoot-stack.sh

Вручную то же самое:

  cd chatwoot && git pull && cd ..
  docker compose build rails
  docker compose build sidekiq
  docker compose up -d rails sidekiq

Репозиторий по умолчанию: https://github.com/qasque/chatwoot-custom.git
Другой URL: CHATWOOT_GIT_URL=https://github.com/you/fork.git ./scripts/clone-chatwoot.sh

Без локальной папки chatwoot/docker/Dockerfile сборка rails не запустится.

Ошибка «pull access denied … repository does not exist»:
  образ qasque-chatwoot:local есть только после сборки. Сначала:
 docker compose build rails && docker compose build sidekiq
  не запускайте up до успешного build. Имя образа в compose должно быть qasque-chatwoot:local (не опечатка qosque-chatbot).

«Зависание» на шаге ~27 (rake assets:precompile / Vite transforming):
  это нормально — самый тяжёлый этап. На слабом CPU/RAM ждите до часа. Смотреть прогресс:
 docker compose build --progress=plain rails

Ошибка: vite build … Command was killed with SIGKILL / ERR_PNPM … SIGKILL:
  это почти всегда OOM (нехватка RAM при «rendering chunks»). Проверка: dmesg -T | grep -i 'killed process'
  Решение 1 (лучшее на VPS): добавить swap, например 4G на Debian:
   sudo fallocate -l 4G /swapfile && sudo chmod 600 /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile
   echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
  Решение 2: в .env задать меньший heap и пересобрать:
   CHATWOOT_BUILD_NODE_OPTIONS=--max-old-space-size=1536 --openssl-legacy-provider
  Решение 3: собрать образ на мощной машине: scripts/build-chatwoot-image.ps1 + docker load на сервере.

Если раньше использовали только GHCR: после git pull test-chat выполните ./scripts/clone-chatwoot.sh,
затем docker compose build rails && docker compose build sidekiq && docker compose up -d rails sidekiq.

Чтобы снова использовать только готовый образ без сборки на сервере:
 в docker-compose.yml у rails/sidekiq уберите build, оставьте image: ghcr.io/qasque/chatwoot-custom:develop
